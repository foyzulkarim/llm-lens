## Summary

Implement API key-based authentication middleware using the `IAuthProvider` interface. The middleware extracts an API key from the `X-API-Key` request header, validates it against the database, and attaches user context to the request for downstream handlers.

## Key Components

- **Auth middleware** — reads `X-API-Key`, calls `IAuthProvider.validateKey`, attaches `req.user`
- **ApiKeyAuthProvider** — implements `IAuthProvider`; backed by `IApiKeyRepo`
- **IApiKeyRepo** + **PrismaApiKeyRepository** — repository interface and Prisma-backed implementation
- **TypeScript declaration merging** — extends Express `Request` with `user?: UserContext`

## Acceptance Criteria

| Condition                      | Expected Behavior              |
| ------------------------------ | ------------------------------ |
| No `X-API-Key` header          | 401, `MISSING_API_KEY`         |
| Empty string as API key        | 401, `MISSING_API_KEY`         |
| Key not found in DB            | 401, `INVALID_API_KEY`         |
| Key found but `isActive=false` | 401, `INVALID_API_KEY`         |
| Key found and active           | Passes through, `req.user` set |
| DB error during validation     | 500, generic message to client |

## Key Decisions

- Header: `X-API-Key` (not Bearer token) — curl-friendly
- Key format: `oui-${uuid}` (e.g. `oui-a1b2c3d4-...`)
- No key caching — SQLite is local, lookups are fast
- Empty/whitespace key treated as missing (trim before lookup)
