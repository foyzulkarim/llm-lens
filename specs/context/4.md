## Task Title

Feature 2: Auth Middleware — API key-based authentication middleware for validating requests against the database

## Summary

Implement API key-based authentication middleware using the `IAuthProvider` interface. The middleware extracts an API key from the `X-API-Key` request header, validates it against the database, and attaches user context (`userId`, `userName`) to the request for downstream handlers. Also implement `ApiKeyAuthProvider` (the concrete `IAuthProvider`) backed by the `ApiKey` Prisma model.

## Key Components

1. **Auth middleware** — Express middleware: extract header → validate → attach user → `next()`
2. **`ApiKeyAuthProvider`** — Implements `IAuthProvider`, backed by `IApiKeyRepo`
3. **`IApiKeyRepo` interface** — `findByKey()` and `create()` methods
4. **`PrismaApiKeyRepository`** — Concrete implementation using Prisma
5. **TypeScript declaration merging** — Extends `Express.Request` with `user?: UserContext`

## Error Responses

| Condition              | Response                       |
| ---------------------- | ------------------------------ |
| No API key / empty key | `401` — `MISSING_API_KEY`      |
| Invalid/inactive key   | `401` — `INVALID_API_KEY`      |
| Valid key              | Passes through, `req.user` set |

## Scope

- **In scope:** Auth middleware, `ApiKeyAuthProvider`, `IApiKeyRepo`, `PrismaApiKeyRepository`, TypeScript types, unit/integration/API tests
- **Out of scope:** Key rotation/expiration, RBAC, rate limiting, admin key management endpoints
